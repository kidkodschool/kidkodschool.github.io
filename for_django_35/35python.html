Обзор

Вам необходимо создать приложение для определения пользовательских задач.

Пользователь будет иметь возможность добавлять, удалять и изменять поставленные задачи.

Цели

  1. Создание базового функционала добавления, изменения, удаления для задач
  2. Определение визуального стиля, с использованием либо CSS-фреймворков, либо собственных стилей.

Технические подробности

Задача реализуется с помощью python не ниже версии 3.3 и django версии не ниже 3.2. CSS фреймворк вы можете выбрать любого тип и версии.

  1. Реализовать внешний вид главной страницы, с отображением всех добавленных задач в приложении с возможностью перехода в выбранную задачу для ее более детального изучения
  2. Добавить CRUD для задач
  3. Реализовать возможность удаления всех завершенных задач в приложении
  4. Максимальное число отображаемых задач на одном экране - 4 (четыре)
  5. Программа должна быть расширяема (с возможностью будущего обновления функционала)

Ключевые этапы - Создание базового функционала приложения

  1. Функционал главной страницы - Когда нет задач, отображение формы для добавления задач

  1. Функционал главной страницы - Когда есть задачи, отобразить форму для добавления задач и созданную задачу (ниже представлен внешний вид для задачи)

  1. Функционал задачи - При нажатии на “Сделано” вы переводите задачу в режим “завершенной”. После этого вы сможете вернуть задачу в режим “Не сделано”, нажав на соответствующую кнопку.

  1. Функционал главной страницы - После добавления “Сделанных” задач, у нас появится новый функционал - Удаление всех завершенных задач.

  1. Функционал задачи - Удалении всех завершенных задач, отобразить удаляемые задачи

  1. Функционал задачи - При нажатии на “Изменить” мы сможем поменять весь контент в задаче или только ее часть.

  1. Функционал задачи - При нажатии на “Х” (удалить) мы можем удалить выбранную задачу.

  1. Функционал задачи - При Удалении/Изменении задачи, перенаправить на страницу с подтверждением этих действий

  1. Функционал главной страницы - максимальное количество задач на одном экране: 4.

  1. Функционал главной страницы - При создании в приложении больше чем 4 задач появится “постраничное отображение” - pagination

  1. Функционал главной страницы - Отображение задач в порядке их добавления
